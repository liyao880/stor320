Step 3: Now to get the rates (part c), we pipe cases into mutate. The mutate function goes through cases row by row. For each row, mutate takes the value of count, which represents the number of cases of the disease, multiplies it by 10,000, and divides that by the value of count in the populations table that has the same row index.

rates = cases %>% mutate(type = "rate", count=10000*count/populations$count)

To explain another way, we take the first row of cases (Afghanistan 1999), get the count (745), multiply by 10,000, get the first row of population (again Afghanistan 1999), get the count (19,987,071), and divide.

This only works because we know that the cases and populations have the same ordering.

Step 4: Now we store it back in the appropriate place (part d). To match the format of table2, we should have “rate” in the type column, and the computed rate in the count column (although this is no longer really a count). We simply add these rows to table2 using the rbind function, and then re-arrange to match the original format:

Exercise 4

preg <- tribble(
    ~pregnant, ~male, ~female,
    "yes",     NA,    10,
    "no",      20,    12
)

This tibble represents numbers of males and females who are or are not pregnant. According to the principles of tidy data, each variable needs its own column, and each observation needs its own row. In this case, the column names male and female represent the values of a variable (call it sex). We need to pivot_longer(gather) these into a sex variable.

Another way to explain it: We’re measuring counts of people. These measurements are split over the columns male and female. In tidy data, one measurement should not split over columns. So, we need to use pivot_longer(gather) to make male and female values of a sex variable, and the corresponding counts all in one column, called count.

Solution 1:

preg %>% pivot_longer(male:female, names_to="sex", values_to="count")
## # A tibble: 4 x 3
##   pregnant sex    count
##   <chr>    <chr>  <dbl>
## 1 yes      male      NA
## 2 yes      female    10
## 3 no       male      20
## 4 no       female    12

Solution 2:

preg %>% gather(male, female, key="sex", value="count")
## # A tibble: 4 x 3

##   <chr>    <chr>  <dbl>
## 1 yes      male      NA
## 2 no       male      20
## 3 yes      female    10

Exercise 2

Very occasionnaly, NA values may represent data carried forward, like writing ditto marks. fill replaces these NAs with the correct repeated value. direction is used to specify whether NAs should be filled using the preceding value or the next value. If an NA represents that the precededing value is repeated (this is the usual way) we use direction="down". If the NA represents a repetition of the next non-NA value (this is rare), we use direction="up".
